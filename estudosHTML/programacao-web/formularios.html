Formularios

Tag usada para capturar, armazenar e retornar dados de usuariso

Os formularios tabem conhecidos como dados de entradas ou de interação com o site (area de login ou area de cadastro) estremamente ligado ao Backend e Banco de dados

Todo formulario começa com a tag (form) e dentra dela tem uma propriedade action que recebera uma logica do backend para receber os dados capturados no formulario

Tambem na tag forms existe um atributo chamado method que recebe um valor podendo ser GET ou POST dependendo do tipo de requisição o formulario sera

A tag input serve para enviar um dado de entrada e o atributo type descreve o tipo de entrada que sera capturado no imput, por exemplo um text ou texto ele abre um campo para ser digitado alguma coisa, nessa tag existe tambem outra propriedade chamada placeholder e uma especie de marca dagua que indica o que o usuario deve fazer

Associado a tag imput usamos a tag label que serve para descrever o que e o campo ou dar sentido a ele, o label tambem pode ser associado diretamente ao seu input nesse caso e so colocar um id com um nome no imput

e colocar esse nome na propriedade for do label, fazendo isso quendo o label for clicado sera direcionado para o input





Existe outros tipos de input como do tipo senha que recebe como valor na propriedade type um password um caracteristica desse campo e que ele esconde os numeros digitados com bolinhas



Outro campo muito usado e o campo de Email, para isso e preciso informar o valor da propriedade type como "email" dentro da tag input


Para tornar o preenchimento desses campos obrigatorios e preciso colocar uma propriedade chamada require na tag input que quer dizer obrigatorio

Outro input muito importante e o Submit que quer dizer enviar serve para enviar os dados recebidos para o backend ou um serivdor mas para isso e preciso informar no campo type que e do tipo submit,tambem existe uma propriedade que se chama value ela serve para nomear esse botao


Inputs para multipla escolha

Input do tipo radio são muito usados para multiplas escolhas (podendo escolher so uma entre muitas), pra isso na propriedade type recebe o valor radio, como e uma tag usada para multiplas escolhas todas as opçoes de escolhas devem receber o mesmo valor na propriedade nome e na propriedade value atibuimos um nome que sera atibuido no backend, associado ao radio devemos colocar um label com id para indicar quel opção o radio se refere
